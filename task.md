### 已知限制

1. **运行时依赖**：需要 Chrome 以 "--remote-debugging-port=9222" 启动
2. **错误处理**："doctor" 和 "ask" 在 provider tab/browser state 不可用时返回结构化运行时错误
3. **真实网页验证前提**：若未打开可复用 provider tab 或未连接到可调试 Chrome，会走结构化错误路径而非真实对话路径

### 后续建议

1. 考虑添加 daemon 模式以减少 Chrome 连接开销
2. 为真实浏览器 smoke test 补充自动化验证夹具
3. 扩展更多 provider 适配器
