# 扩展会话转录层 Post-Regression 计划（2026-04-13）

## 背景

当前 `feature/extension-session-layer-mvp` 已完成：

- 统一发送链路：转录去重稳定（Gemini 不再重复写碎片；Grok 不再写入 prompt echo）
- DeepSeek/Grok 完成态：`responding -> completed` 可收敛
- 手动续聊：在 provider iframe 内直接发送也能入账（user/assistant turn + 状态）

但仍有 2 类体验问题需要补齐：

1. Dashboard 右侧 `Provider 原始记录` 展开后会在刷新/轮询时折叠（影响可用性）。
2. 记录呈现偏“消息流”，缺少“对话式（一问一答）”视图；同时 Gemini 初始欢迎语是否展示需要明确策略。

## 目标与范围

目标：

- 提升 Dashboard transcript 面板交互稳定性与可读性。
- 不改变现有会话模型、持久化策略与发送链路。

非目标：

- 不做 provider 原生历史列表适配。
- 不做 CLI、结构化摘要、搜索、导出等能力（除非另立任务）。

## 任务拆分（建议执行顺序）

### T-20260413-011（P0）Dashboard 原始记录展开态稳定

- 内容：修复 `Provider 原始记录` 展开后被刷新折叠。
- 验收：展开任意 provider 的原始记录，等待 2 次轮询刷新后仍保持展开；用户点击展开/折叠不闪回。

### T-20260413-012（P1）Gemini 欢迎语策略

- 内容：对 `需要我为你做些什么？` 这类欢迎语做 UI 策略（隐藏/折叠/标注 system）。
- 验收：用户未发送前，Gemini 欢迎语不干扰主时间线；仍可在需要时查看（不丢数据）。

### T-20260413-013（P2）对话式展示（可选）

- 内容：新增“对话式视图”，把同 provider 的 `user -> assistant` 尽量配对展示。
- 验收：时间线可切换为对话式视图；同轮问答相邻展示；不影响现有“消息流”视图。

### T-20260413-014（P0）Task7 二次回归收口

- 内容：在一条新会话中重跑：新建会话 -> 统一发送 -> 手动续聊 -> 恢复会话 -> 核对 transcript/timeline。
- 验收：四条链路均可复现成功；把相关任务状态流转收口到 `待确认/完成`。
