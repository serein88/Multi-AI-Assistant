# code-review-graph 使用指南

## 简介

**code-review-graph** 是一个开源的代码上下文优化工具，通过建立代码库的结构图谱，让 AI 编程工具只读取与当前任务真正相关的文件，从而大幅降低 token 消耗。

### 核心指标

| 指标 | 数据 |
|------|------|
| 平均 token 节省 | 8.2x |
| 最高 token 节省 | 49x |
| 语言支持 | 23+ 种 |
| AI 平台支持 | 11 个 |
| 漏报率 | 0% (100% Recall) |
| 增量更新速度 | < 2 秒（3000 文件项目） |

### 核心特性

- **100% 本地运行**：代码不出域，无需外部数据库，无云服务
- **精确影响分析**：基于 AST 语法树和调用关系图谱追踪代码变更影响范围
- **MCP 协议集成**：通过 Model Context Protocol 接入主流 AI 编程工具

---

## 核心概念

### Blast Radius（爆炸半径）

**Blast Radius** 是该工具最核心的概念，指的是一次代码变更会影响到哪些文件。

**工作示例**：假设修改了 `authentication` 模块中的 `verify_token` 函数，MCP 服务会：

1. 找到所有调用这个函数的文件
2. 找到所有依赖这个模块的文件
3. 找到所有测试这个行为的测试用例

最终交付给 AI 的只有 blast radius 内的文件清单。

### 安全策略

工具采用 **宁可多报、也不漏报** 的策略。虽然会产生 false positive（误报），但确保所有真正受影响的文件都被包含，避免遗漏关键依赖。

## 工作原理

### 技术架构

```
代码文件 → Tree-sitter 解析 → AST 语法树 → 图谱建模 → MCP 服务 → AI 工具
```

### 1. 代码解析

使用 **Tree-sitter**（老牌代码 AST 解析库）将任意语言的代码解析成语法树，从中提取：

- 函数和类的定义
- 调用关系
- 继承关系
- 导入依赖
- 测试覆盖

### 2. 图谱存储

将整个代码库建模为一张图：

| 节点类型 | 边类型 |
|----------|--------|
| 函数 | 调用关系 |
| 类 | 继承关系 |
| 模块 | 导入依赖 |

所有数据存入 **SQLite 数据库**（本地存储）。

### 3. MCP 协议接入

通过 **MCP (Model Context Protocol)** 协议接入 AI 编程工具，当执行代码审查时：

1. MCP 服务分析 blast radius
2. 追踪 color dependent 和相关测试
3. 只把真正相关的文件交给 AI

### 4. 超精简上下文交付

核心工具 `get_minimize_context_to` 只使用约 **100 tokens** 的超精简上下文。

## 核心模块解析

### parser.py - 多语言解析器

- 依赖 **Tree-sitter**
- 支持 23+ 种编程语言的 AST 解析

### graph.py - 图存储与遍历

- 基于 **SQLite** 存储图结构
- **BFS 遍历**（广度优先搜索）
- **Blast Radius 计算**

### incremental.py - 增量更新

- 通过 Git hook 监听 **SHA-256 哈希**变化
- 差异检测：只重解析改过的文件
- 保持图谱实时更新

### toolset.py - MCP 工具集

提供 **28 个 MCP 工具**，包括：

| 工具名 | 功能 |
|--------|------|
| `get_impact_radius` | 获取影响范围 |
| `detect_changes` | 检测代码变更 |
| `query_graph` | 图谱查询 |
| `semantic_search` | 语义搜索 |
| `hub_nodes` | 查找中心节点 |
| `bridge_nodes` | 查找桥接节点 |
| `knowledge_gaps` | 发现知识盲区 |

### 可选扩展模块

- **向量嵌入**：支持语义搜索
- **D3.js 生成**：交互式可视化视图
- **Wiki 自动生成**：架构文档自动化
- **社区检测**：识别代码模块边界
- **风险评分**：评估变更风险等级

## 安装与配置

### 安装步骤

```bash
# 1. 安装核心包
pip install code-review-graph

# 2. 初始化配置（自动检测 AI 平台并写入 MCP 配置）
code-review-graph install

# 3. 多仓库监控（可选）
# 使用 Claude Daemon 守护进程后台监控多个项目
```

### 安装特性

- 一步到位安装
- 自动检测用户使用的 AI 编程平台
- 自动写入对应 MCP 配置

## 应用场景

### 场景一：代码审查（PR Review）

**操作**：PR 改了 50 个文件

**结果**：一键生成影响分析报告，高风险/中风险文件一目了然

**操作**：接手 1 万行代码的项目

**结果**：运行 `vki` 命令，自动生成：
- 架构文档
- 可视化图谱
- 调用链路图

**操作**：想动一个上古雷（legacy code）

**结果**：使用 `find_nodes` 找出高连接节点，评估影响范围后再动手
